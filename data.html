Each group member independently evaluated the prototype using Nielsen's 10 Usability Heuristics. Below is a consolidated summary of all findings.

H1

Visibility of System Status Low Severity

System keeps users informed through appropriate feedback.

✅ Progress rings, streak badges, and checkmarks clearly show status at all times.
H2

Match Between System and Real World Low Severity

Use words and concepts familiar to the user.

✅ Labels use everyday language — "Drink Water", "Workout", "Bed time". Emoji icons reinforce meaning.
H3

User Control and Freedom Medium Severity

Clearly marked emergency exits for mistaken actions.

⚠️ Skip and Cancel buttons exist but no undo for deleted habits — a gap found by 3 evaluators.
H4

Consistency and Standards Low Severity

Same words, situations, and actions mean the same thing.

✅ Green header, white cards, and bottom nav are consistent across all 8 screens.
H5

Error Prevention Medium Severity

Careful design prevents problems from occurring.

⚠️ Placeholder hints help, but no confirmation dialog before deleting tasks or habits.
H6

Recognition Rather Than Recall Low Severity

Minimize memory load by making options visible.

✅ All icons paired with text labels. No icon-only elements found throughout the prototype.
H7

Flexibility and Efficiency of Use Medium Severity

Accelerators for expert users.

⚠️ Quick Actions row helps. However, no swipe gestures available for power users.
H8

Aesthetic and Minimalist Design Low Severity

No irrelevant or rarely needed information.

✅ Each screen focuses on one task. Clean card layouts with ample whitespace.
H9

Help Users Recognize and Recover from Errors High Severity

Error messages in plain language that suggest solutions.

🔴 No error states or validation messages designed for empty forms. Most cited issue by all 5 evaluators.
H10

Help and Documentation High Severity

Help may be necessary even for well-designed systems.

🔴 No onboarding tutorial or tooltip guidance. Intensity slider and streak dots confuse new users.
👩

Jodelyn – Screen 5: Mood Tracker & Diary

Focus: mood selection, diary entry, past entries, daily check-in flow.

H1

Visibility of System Status Pass

The 5-day streak badge and blinking cursor in the diary field clearly indicate active state to the user.
H9

Error Recovery Fail

No validation shown when saving without selecting a mood. Recommendation: Add prompt "Please select your mood first."
H10

Help & Documentation Fail

Intensity slider has no tooltip explaining what "Moderate" means. First-time users may be confused.
👨

Julius – Screen 6: Home Dashboard

Focus: dashboard summary, quick stats, navigation, overall layout.

H4

Consistency & Standards Pass

All 8 screens maintain consistent green header, card style, and bottom navigation. Excellent visual consistency.
H7

Flexibility of Use Partial

Quick Actions row helps returning users. No swipe gestures for power users available.
H3

User Control & Freedom Fail

Duplicate progress rings in original dashboard caused confusion. No undo for dismissed notifications.
👩

Celes – Screen 3: Daily Planner

Focus: to-do list, schedule, task management, planner layout.

H2

Match with Real World Pass

Task names, times, and labels use familiar language. "Breakfast", "Gym", "Bed time" are instantly understood.
H5

Error Prevention Partial

No confirmation dialog when deleting a task. Recommendation: Add undo toast "Task deleted. Undo?"
H8

Aesthetic Design Pass

After redesign with greeting and progress bar, the layout balance improved significantly.
👨

Meljun – Screen 4: Habit Tracker

Focus: habit list, reminders, day selector, progress tracking.

H1

System Status Visibility Pass

Checkmarks, progress bars, and streak badges all clearly communicate completion state.
H3

User Control & Freedom Partial

Reminders can be toggled on/off. However, no undo after accidentally marking a habit done.
H10

Help & Documentation Fail

Streak dot colors (green/yellow/grey) are not self-explanatory. A color legend is needed.
👨

Mark – Screen 7: Analytics

Focus: analytics charts, trend data, and data presentation.

H6

Recognition over Recall Pass

Charts include labeled axes and color legends. Users do not need to memorize representations.
H7

Flexibility of Use Partial

Day/Week/Month toggle present but no data export or filter by specific habit type.
H9

Error Recovery Fail

Empty chart shows blank space with no explanation. Needs empty state: "No data logged yet. Start tracking!"

Usability Testing Survey Instrument

Respondents interacted with the Figma prototype then answered these questions using a 5-point Likert scale (1 = Strongly Disagree, 5 = Strongly Agree).

Q1

The app was easy to navigate and understand.

Category: Learnability

Q2

The design is visually appealing and consistent.

Category: Aesthetics

Q3

I can clearly understand what each screen is for.

Category: Clarity

Q4

The app would help me manage my daily wellness routine.

Category: Utility

Q5

I encountered no confusing elements during testing.

Category: Error-Free Experience

Q6

I would recommend this app to others.

Category: Satisfaction
